# Hackathon DevConnect – Projeto FlixDev

📍 **Instituição:** SENAI Bahia  
🏫 **Unidade:** IFTP – Instituto de Formação Técnica Profissional – Candeias  
💻 **Curso:** Desenvolvimento de Sistemas  
🎯 **Tipo de Atividade:** Avaliação Prática Integrada (Hackathon Educacional)

---

## Sobre o Projeto

Este repositório documenta um modelo de avaliação prática aplicado no **IFTP – SENAI Candeias**, desenvolvido para estudantes do curso de **Desenvolvimento de Sistemas**.

A atividade foi estruturada no formato de **Hackathon educacional**, simulando um ambiente corporativo real de desenvolvimento de software. Durante o desafio, os alunos são organizados em equipes que trabalham em etapas diferentes de um mesmo sistema backend, utilizando um fluxo de **handoff**, prática comum em empresas de tecnologia onde um time inicia o desenvolvimento e outro time continua a evolução e manutenção do sistema.

O projeto **FlixDev** simula a construção e manutenção de um pequeno sistema de catálogo e gestão de assinantes, utilizando conceitos fundamentais de programação e integração com APIs.

---

## Objetivo Didático

O Hackathon foi desenvolvido com o objetivo de proporcionar aos estudantes uma experiência prática que estimule:

- Desenvolvimento de **lógica de programação em JavaScript**
- Manipulação de **arrays e estruturas de repetição**
- Utilização de **métodos nativos da linguagem**
- Compreensão de **requisições HTTP (GET, POST, PUT, DELETE)**
- Interação com **APIs simuladas**
- Escrita de **documentação técnica**
- Leitura e manutenção de código produzido por outros desenvolvedores
- Organização e colaboração em **trabalho em equipe**

---

## Proposta Pedagógica

A metodologia busca aproximar os estudantes de **cenários reais de desenvolvimento de software**, incentivando:

- pensamento lógico
- autonomia na resolução de problemas
- clareza na documentação
- responsabilidade coletiva sobre o projeto

Durante a atividade, os alunos experimentam um fluxo semelhante ao encontrado em ambientes profissionais, onde a **qualidade da documentação e do código impacta diretamente o trabalho da equipe seguinte**.

---

## Estrutura da Atividade

O Hackathon é dividido em **duas etapas principais**:

### Dia 1 – Equipe A
Responsável pela criação da base do sistema:

- Estrutura inicial do catálogo em JavaScript
- Organização das categorias
- Criação da API simulada
- Registro de assinantes via requisições HTTP
- Documentação do funcionamento do sistema

### Dia 2 – Equipe B
Responsável pela evolução e manutenção do sistema:

- Expansão do catálogo
- Inserção de novas categorias
- Atualização de registros na API
- Exclusão de usuários
- Teste e análise de erros de requisição HTTP

---

## Tecnologias Utilizadas

Durante a atividade são utilizados os seguintes recursos:

- **JavaScript**
- **Node.js (execução em terminal)**
- **MockAPI**
- **Postman**
- **JSON**
- **GitHub (documentação do projeto)**
